# SouqAgent Pay Final Submission

## Title

SouqAgent Pay

## Short Description

SouqAgent Pay is an AI spending desk for UAE/GCC SMEs where autonomous agents discover paid services, obey owner-defined spend policy, authorize stablecoin payments, fund Arc USDC escrow, submit deliverable proof through Circle Wallets, and release settlement with a shareable receipt trail.

## What It Does

SouqAgent Pay lets a business owner set a spend cap and ask an AI agent to verify a supplier before approving settlement. The agent discovers a paid supplier-risk API, handles the x402-style payment challenge, receives risk data, funds an Arc USDC escrow job, stores a durable task ledger in Supabase, accepts deliverable proof, submits that proof onchain through Circle Wallets, and releases the escrow once proof exists.

## What Makes It Real

- The deployed app runs in `testnet` mode.
- The escrow contract is deployed on Arc Testnet.
- Production API calls create and fund real Arc escrow jobs.
- Circle Wallets submit deliverable proof to the Arc contract from the agent/seller wallet.
- Supabase stores durable task, policy, deliverable, and receipt records.
- Owner access protects spend and settlement controls.
- The production status panel shows live deployer, escrow, owner wallet, and agent wallet balances plus the latest Arc job.
- Receipt pages are public and shareable for audit/review.

Core flow:
1. Owner enters an instruction and spend cap in the SouqAgent Pay UI.
2. The backend checks owner access and applies the budget policy.
3. The agent calls a paid seller API and handles the x402-style payment challenge.
4. The result is attached to the task ledger with payment receipt evidence.
5. The backend funds an Arc Testnet USDC escrow job.
6. Circle Wallets submit deliverable proof onchain from the agent/seller wallet.
7. The escrow can be released only after deliverable proof exists.
8. The receipt page shows policy, payment, Arc escrow, deliverable tx, and release tx.

## Circle Product Feedback

### Why These Products

SouqAgent Pay needs programmable stablecoin money that agents can use without exposing unlimited funds or private keys. USDC provides the settlement asset, Arc provides deterministic USDC escrow, Circle Wallets let the agent/seller submit contract actions safely, and Gateway/Nanopayments/x402 fit the paid API model.

### What Worked Well

- The x402 flow is easy to explain and demo.
- Arc's EVM compatibility made the escrow workflow straightforward.
- Circle Wallet contract execution allowed deliverable proof submission without exposing the seller/agent private key.
- USDC-denominated settlement is understandable for business owners.

### What Could Improve

- A single Arc + Wallets + Gateway + Nanopayments starter would help hackathon teams move faster.
- More complete buyer/seller x402 sample apps would reduce integration guesswork.
- Circle Wallet contract execution can return a Circle transaction id before a chain tx hash; a documented polling recipe would improve receipt UX.

### Recommendations

- Publish a "paid API in 15 minutes" starter.
- Provide a canonical milestone escrow example on Arc.
- Provide sample receipt components for Gateway/Nanopayments and Circle Wallet contract execution.
